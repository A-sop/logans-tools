# GABC Website Redo — Stakeholder Workshop Notes (Draft)

This document captures the decisions the board needs to make to unblock the build.

## 1) Primary CTA (homepage)

Pick **one** primary CTA for Phase 1:

- **Option A: “Become a member”** (membership-led growth)
  - Pros: directly drives revenue + long-term network growth
  - Cons: some visitors want to “test” via an event first
- **Option B: “View upcoming events”** (events-led growth)
  - Pros: lower friction; first touchpoint for new people
  - Cons: membership conversion needs stronger follow-up journey

**Recommendation (default):** Option B (events-first), with a prominent secondary CTA for membership.

## 2) Languages

- **Phase 1 default:** English-only (fastest)
- **If German is required:** define which pages must be bilingual (Home + Membership + Legal are common)

## 3) Membership tiers (public vs gated)

Decide what is visible publicly:

- **Public:** tier names, benefits, pricing (if any), and how to apply
- **Gated:** member directory, member-only content library, discounted tickets

## 4) Member portal scope (yes/no)

- **Phase 1:** marketing site only (recommended for board approval)
- **Phase 2+:** portal for privileged content + purchases

**Decision needed:** Portal in the initial build plan or explicitly deferred until Phase 2.

## 5) Focus areas (member preferences)

Define the initial taxonomy shape:

- **Chapters:** Frankfurt, Berlin, Munich (yes/no)
- **Formats:** in-person, online (yes/no)
- **Themes/industries:** board-defined list (e.g. renewables, tech, manufacturing)

**Rule:** council-wide announcements always bypass focus areas.

## 6) Payments (Stripe + WERO)

- **Stripe:** required for recurring membership + tickets + documents
- **WERO:** confirm whether “must-have for launch” or “Phase 4 if feasible”

## 7) Content operations

Who owns ongoing updates?

- events owner(s)
- membership owner
- editor(s) for posts/news
- legal approver

If editors are non-technical, pick a headless CMS in Phase 2.
